# Codex Skills Best Practice

Create a skill when a workflow repeats enough that you can name it.

Principles:

- keep one skill focused on one job
- write the description as a trigger boundary
- start instruction-only
- add scripts only when deterministic behavior matters
- add references when progressive disclosure improves reliability

Common mistake:

- writing the description like a summary instead of “when should Codex use this”

Good signal:

- if you keep pasting the same long prompt, the workflow probably wants to become a skill
