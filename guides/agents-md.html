AGENTS.md guide

Keep it short, operational, and true.

Codex reads AGENTS.md before it starts work. That makes it the right place for durable repo expectations, not a dumping ground for everything your team believes about software.

What belongs in the file

  • How to run, build, lint, and test the repo
  • Key directories and ownership boundaries
  • Constraints, do-not rules, and verification standards
  • Review expectations and what “done” means

What usually does not

  • Vague style preferences with no examples
  • Long architectural essays better kept in separate docs
  • Rules your team does not actually enforce
  • Temporary task instructions that should live in the prompt
# AGENTS.md

## Repo expectations

- Run `pnpm lint` and `pnpm test` after changes.
- Keep edits scoped to the requested feature or fix.
- Do not add new dependencies without approval.
- Update docs when public behavior changes.

## Done when

- Tests pass locally
- The requested behavior is verified
- Risky tradeoffs are called out explicitly

Layering model

Use a global file for personal defaults, a repo-level file for shared standards, and nested overrides only where local rules truly differ. Keep the more specific file close to the work.
