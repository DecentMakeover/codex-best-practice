Concepts

Plugins .codex-plugin/plugin.json Distribution format for shipping skills, apps, and MCP together.
Apps / Connectors /apps, [apps.*] Connected tools from ChatGPT surfaced inside Codex for work that crosses systems.
Automations Codex app Background recurring tasks once a workflow is stable enough to trust on a schedule.
Review /review, diff view Review loops turn Codex from a fast generator into a more reliable engineering tool.
Workflows AGENTS.md + config + skills + agents Codex orchestration is built from durable repo files plus built-in commands.

Feature Why it matters

Automations Stable workflows can move into background runs instead of living in your todo list.

Orchestration workflow

The Claude reference repo leans on Command → Agent → Skill. Codex’s closer equivalent is: 

Prompt or /plan
      ↓
AGENTS.md + .codex/config.toml
      ↓
Subagent role
      ↓
Skill
      ↓
Code or report
      ↓
/review + diff + commit

How to use

codex
> /plan Inspect this repo and tell me which AGENTS.md, config, skills, and agents I should copy into my own project first.
  1. Copy AGENTS.md and rewrite it for your repo.
  2. Copy .codex/config.toml and trim it to the defaults your team will actually use.
  3. Add one skill for a workflow you already repeat.
  4. Add one read-only subagent role for research or review.
  5. Add MCP only when you are clearly losing time to copy-paste.

Tips

Planning

  • Start with /plan for ambiguous or multi-step work.
  • Put goal, context, constraints, and done-when in the first prompt.
  • Keep one thread per coherent task.

AGENTS.md / Config

  • Keep AGENTS.md short and operational.
  • Put deterministic defaults in config.toml, not prose.
  • workspace-write + on-request is a strong default.

Skills / Subagents

  • Create a skill when a workflow repeats enough to notice the pattern.
  • Write the skill description as a trigger boundary.
  • Delegate bounded sidecar work, not the whole problem.

Review / MCP

  • Add live tools only when copy-paste is becoming the bottleneck.
  • openaiDeveloperDocs is a strong first MCP server.
  • Use /review before commit.
