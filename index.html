Operating model

  1. Prompt with goal, context, constraints, done-when.
  2. Move recurring rules out of the prompt and into files.
  3. Make reliable workflows discoverable and shareable.
  4. Keep external context live instead of pasted.

Workflow ladder

Recommended order of operations.

01

Start with prompts and plan mode

Use clear task framing first. If the work is ambiguous, ask Codex to plan before it edits anything.

02

Encode the rules you repeat

Move durable expectations into AGENTS.md and your default config so they stop living in chat history.

03

Package stable behavior

Convert reliable flows into skills, add MCP when context needs to stay live, and distribute with plugins when the workflow is ready for others.

Daily command set

/plan /init /status /mcp /apps /review /diff /resume /fork

What to keep in the repo

The landing page explains the system. The repo itself should contain working examples of that system: a real AGENTS.md, a real .codex/config.toml, and at least one useful skill.

Avoid this trap

Do not document fantasy workflows.

If the page says “use skills,” the repo should contain a real skill. If it says “review with Codex,” show the command and the review expectations. Keep the landing page tied to working artifacts.
